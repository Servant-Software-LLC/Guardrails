Guardrail — verified AFTER the task's action; must pass for the task to finish

Edge direction — edges run in execution order, from a dependency to its dependent (A → B = B dependsOn A). A cyan mid-edge arrow marks each edge's direction, so a long edge routing PAST an unrelated box is never a dependency on it — follow the arrow to its real target.
