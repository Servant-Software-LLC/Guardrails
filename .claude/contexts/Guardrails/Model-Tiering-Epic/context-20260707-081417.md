- Consulted the agent team — guardrails-architect, guardrails-devils-advocate, and a
  general-purpose agent doing independent external market research — run in parallel (no shared
  context between them) on whether #201's provider-routing system should become a standalone OSS
  repo other AI projects could consume. All three converged independently:
  - Architecturally separable: registry, budget probes, the resolution function, escalation
    mechanics, and interactive steering are generic; WHO assigns the difficulty tag (plan-breakdown/
    task.json-coupled) and WHAT triggers escalation (TaskExecutor's retry-loop-coupled) are
    Guardrails-specific.
  - BUT the "deterministic gate makes weak models safe" thesis — Guardrails' entire justification
    for routing risky work to a cheap/local model — does NOT extract; it lives in TaskExecutor's
    guardrail-verdict machinery, unrelated to the router, so an external consumer without an
    equivalent gate inherits full risk. Minus that thesis, the differentiation shrinks to a UX/
    config layer Portkey's existing conditional-routing/webhook primitives could approximate.
  - Adoption odds estimated 2/5 (unlikely) per independent market research: real interactive-
    threshold-pause routing doesn't exist elsewhere (a genuine gap), but entrenched incumbents
    (LiteLLM 20k+ stars, funded Portkey) and the "reimplemented not reused" base rate (e.g. vLLM
    built its own semantic router rather than adopt an existing well-matched solo one) argue
    against outside adoption.
  - Devil's-advocate flagged a separate, concrete finding: model tiering/provider management is
    NOT a named v2 bet in docs/plans/03-roadmap.md — the whole epic grew organically off two
    already-shipped tickets (#198, #200) without passing the roadmap's own v2-bet gate.
  - Verdict (unanimous): build in-repo first, behind a narrow caller-agnostic interface (an
    IModelRouter-style `Resolve(tag, escalationLevel, context) -> (provider, model, effort)`,
    mirroring the IPromptRunner seam in PromptRunnerRegistry.cs) so a LATER extraction is
    mechanical if ever warranted — do NOT stand up a separate repo/CI/release cadence now. If
    anything is worth publishing today, it's the design thesis as a write-up, not a redistributable
    library whose safety claim depends on a gate that stays behind.
  - Posted this full verdict (plus the roadmap-gate finding) as a comment on #201.
