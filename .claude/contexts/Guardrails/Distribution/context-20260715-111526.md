BLOCKERS/NOTES:
- Three GitHub Actions gotchas found & fixed this session:
  1. Global `-p:AssemblyName=guardrails` renames EVERY project (Cli + Core) → both become
     guardrails.dll → NETSDK1152 publish collision. Fix: leave assembly names alone, rename
     the apphost post-publish (`mv Guardrails.Cli → guardrails`) in the workflow.
  2. A release created by the built-in GITHUB_TOKEN does NOT emit `release: published`
     (event-chain suppression). bump-tap therefore triggers on `workflow_run` of "Release".
  3. The org creates repos with a `master` default branch; do NOT hardcode PR `base: main`
     (peter-evans/create-pull-request defaults to the tap's checked-out branch when omitted).

  smoke all depend on it. Achieved by post-publish rename, not AssemblyName.
